- **What Part 2 adds**
- Profile viewing with public/private behavior to protect user data
- Follow system with requests and approvals so private users control access
- Followers and following lists to show relationships at a glance
- Basic notifications for follow requests and a welcome message to improve onboarding

- **What new APIs exist (grouped)**
- Profile APIs for viewing a profile and toggling privacy
- Follow APIs for requesting, accepting/declining, listing requests, listing followers/following, and unfollowing

- **How privacy + follows work (high level)**
- Public profiles are visible to anyone
- Private profiles are only fully visible to the owner and accepted followers
- Follow requests stay pending until approved or declined by the target user

- **What is intentionally NOT implemented yet**
- User activity endpoints
- Post, group, event, and chat features beyond the current scope
- Full frontend UX for profiles and follow flows
