# Τι είναι αυτά?

Είναι προγραμματιστικά μοντέλα, ή αλλιώς αρχιτεκτονικά ή λογικά μοντέλα. Δηλαδή τρόποι οργάνωσης της λογικής ενός προγράμματος, όχι προγράμματα ή εργαλεία τα ίδια.

# Τι κάνουν?

## Pipeline

Κάνει **το ένα βήμα μετά το άλλο**, ενώ περνάει το αποτέλεσμα στο επόμενο βήμα, όπως μία **γραμμή παραγωγής** σε εργοστάσιο. 

Κάθε βήμα παίρνει κάποιο input, κάνει κάτι και δίνει το output στο επόμενο βήμα.

Παράδειγμα
Έχουμε το κείμενο "Hello world !" και θέλουμε να γίνει "Hello World!"

Τα βήματα του pipeline θα μπορούσαν να είναι:
1. Remove extra spaces → "Hello world!"
2. Uppercase → "Hello World!"

**Είναι πιο δυνατό σε: (hex), (bin), (up), (low), (cap), a/an**

## FSM (Finite State Machine)

Είναι σαν ένα μικρό ρομποτάκι που **συμπεριφέρεται διαφορετικά ανάλογα σε ποια κατάσταση (state)** βρίσκεται.

Έχει:

- States (e.g., “reading text”, “inside quotes”, “after punctuation”)
- Transitions (rules that say when to move to another state)

Παράδειγμα (text tool):

Θέλουμε να αφαιρέσουμε τα κενά *μόνο* έξω από τα εισαγωγικά.

FSM states:

🟢 OutsideQuotes

🟣 InsideQuotes

Rules:

When you see ", switch states.

When in OutsideQuotes, remove extra spaces.

When in InsideQuotes, keep everything.

Οπότε το FSM μπορεί να "θυμάται" περιεχόμενο, κάτι που ένα απλό pipeline δεν μπορεί.

**Είναι πιο δυνατό σε: Διόρθωση σημείων στίξης, quotes, ..., ?!**

## Επιλογή αρχιτεκτονικής και λόγοι

Για την συγκεκριμένη άσκηση, παρόλο που και οι δύο αρχιτεκτονικές θα ήταν πολύ ενδιαφέρουσες και θα είχαν πολλά να με διδάξουν καθώς επίσης ο συνδιασμός τους ίσως να έκανε πιο εύκολα τα πράγματα, θα επιλέξω να χρησιμοποιήσω την pipeline για τους παρακάτω λόγους:

1. Στο pipeline είναι **πιο εύκολο να ελεγχθεί το κάθε του σημείο ξεχωριστά και να γίνει debugging**. Στη δεδομένη στιγμή δεν έχω πολύ χρόνο στην διάθεσή μου και δεδομένου ότι είμαι πολύ αρχάρια πιστεύω είναι πιο παραγωγικό να χρησιμοποιήσω τη λύση που δίνει πιο εύκολη και άμεση διόρθωση
2. Είναι **πιο απλό** στη δομή του
3. Είναι **εφικτό να διαχειριστεί όλες τις παραμέτρους της άσκησης** και παρόλο που το FSM είναι πιο αποδοτικό και εύκολο σε κάποια κομμάτια (πχ εισαγωγικά), μπορεί και το pipeline να γίνει με σχετικά εύκολο και όχι κοστοβόρο τρόπο.

### Αρνητικά του pipeline

Θα είναι δύσκολο να διαχειριστεί nested quotes ή άλλες πιθανές καταστάσεις που χρειάζεται να "θυμάται" κάτι.
