### what-osint

#### Subject

Understanding the definition and scope of OSINT (Open-Source Intelligence) is crucial for any cyber analyst. This exercise introduces the basic concepts, distinctions, and examples of OSINT in a cybersecurity context.

Your tasks:

- Define what OSINT means in the context of cybersecurity.
- List common sources of OSINT (at least 5).
- Explain the difference between OSINT, SOCMINT, and SIGINT.
- Mention at least 3 common use cases for OSINT in cyber operations.
- Write your response in a well-structured Markdown file named `osint_definition.md`.

Your explanations must be **in your own words**. Do not copy-paste textbook definitions.

#### Deliverables

- A markdown file named `osint_definition.md` containing:
  - A clear definition of OSINT in cyber.
  - A list of OSINT sources.
  - Comparisons between OSINT, SOCMINT, and SIGINT.
  - At least 3 OSINT use cases in cybersecurity.
  


### osint-legal

#### Subject

Understanding the legal boundaries of OSINT is essential to operate responsibly and lawfully.

Your tasks:

- Research the legal frameworks (national or international) that affect OSINT collection.
- Identify at least 3 laws, regulations or articles that limit or authorize OSINT practices.
- Give examples of actions that are considered **illegal**, **borderline**, or **legal** in OSINT context.
- Write your response in a Markdown file named `legal_limits.md`.

#### Deliverables

- A markdown file named `legal_limits.md` including:
  - Legal references for OSINT use.
  - At least 3 legal documents/articles mentioned.
  - Examples for each category: legal / borderline / illegal.

### toolbox

#### Subject

Knowing the essential tools is a foundational part of doing OSINT work effectively.

Your tasks:

- Research and list at least 5 essential OSINT tools.
- For each tool, describe what it does and in which situation it is most useful.
- Include both command-line and web-based tools.
- Write everything in a Markdown file named `osint_tools.md`.

#### Deliverables

- A markdown file named `osint_tools.md` including:
  - At least 5 OSINT tools.
  - Description and use case for each tool.
  - Clear separation between tools, well-formatted.

### osint-ethics

#### Subject

Ethics in OSINT work is as important as legality. This task will help you reflect on the moral responsibilities of an OSINT analyst.

Your tasks:

- Define what ethical OSINT means.
- Discuss the risks of doxxing, harassment, or unintended harm.
- Provide at least 3 ethical principles to follow during OSINT investigations.
- Write your reflection in a Markdown file named `osint_ethics.md`.

#### Deliverables

- A markdown file named `osint_ethics.md` including:
  - A personal definition of ethical OSINT.
  - Risks and examples of unethical OSINT.
  - At least 3 proposed ethical guidelines or rules.
