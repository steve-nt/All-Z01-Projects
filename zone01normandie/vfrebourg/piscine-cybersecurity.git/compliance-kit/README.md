### checklist

#### Goal

Create a legal compliance checklist for a small tech company handling user data.

#### Instructions

Include checks for:

- GDPR compliance (data processing, consent, rights, etc.)
- Cookie and tracker obligations
- Incident response planning
- Data retention and deletion
- Vendor contracts and audits

Make it practical and scannable.

#### Deliverables

- A markdown checklist with clear sections and items.

### doc-pack

#### Goal

Draft base legal documents/templates for compliance.

#### Instructions

Create templates for:

- Privacy Policy
- Data Processing Agreement (DPA)
- Breach Notification Email
- Cookie Consent Notice

They can be partial, but must include standard legal clauses.

#### Deliverables

- A folder with each template as a `.md` file.

### kit-ready

#### Goal

Assemble a full legal starter kit folder.

#### Instructions

You’re helping launch a new product — you need to gather all useful compliance docs in one place for the team.

The kit should include:

- Checklist (from previous exercise)
- Legal document pack (doc-pack)
- Any guidance or onboarding notes for internal teams

Organize it as a structured folder (zip or repo style).

#### Deliverables

- A zipped folder or repo-ready structure with a `README.md` explaining each element.
