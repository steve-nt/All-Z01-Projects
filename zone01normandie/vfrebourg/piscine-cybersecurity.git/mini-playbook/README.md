### template-kit

#### Subject

In this exercise, you will create a basic Incident Response (IR) playbook template tailored for your organization.

Your tasks are:

- Design a clear and structured playbook template that can be reused for different types of incidents (e.g., phishing, ransomware, data leak).
- Include essential sections such as:  
  - Incident description  
  - Initial detection and reporting  
  - Containment steps  
  - Eradication procedures  
  - Recovery actions  
  - Communication plan  
  - Post-incident review and lessons learned
- Ensure the template is easy to follow and can be adapted by various teams.

#### Deliverables

- A markdown file named `IR_playbook_template.md` containing the complete playbook template.


### fill-steps

#### Subject

In this exercise, you will reuse the incident response (IR) playbook template you created earlier to build a specific IR plan for a **phishing attack** scenario.

Your tasks are:

- Start from your `IR_playbook_template.md` or copy its structure into a new file.
- Write a complete playbook named `phishing_playbook.md` detailing the steps and actions to take in response to a targeted phishing email attack.
- Populate **each section** of the template with realistic and well-thought-out actions, decisions, and escalation points.
- Ensure your playbook includes:
  - A clear description of the phishing scenario (e.g., credentials harvesting, malicious attachment)
  - Specific containment actions (e.g., user isolation, email quarantine)
  - Recovery actions (e.g., password resets, user awareness)

#### Deliverables

- A markdown file named `phishing_playbook.md` containing a fully filled playbook based on your earlier template.

### share-doc

Incident response is a team effort — your documentation should be accessible and useful to others.

In this exercise, you’ll simulate sharing your incident response playbook with other members of your organization by producing a **clean, shareable version** of the document.

Your tasks are:

- Take the file `phishing_playbook.md` from the previous exercise.
- Reformat and structure it clearly for internal sharing:
  - Use headers, lists, and consistent formatting.
  - Add a brief introduction and usage note.
  - Include a changelog at the end.
- Save your new version as `shared_phishing_playbook.md`.
- Imagine this document will be uploaded to your internal wiki or incident response toolkit.

#### Deliverables

- A markdown file named `shared_phishing_playbook.md` presenting a clean, clear, and shareable version of the phishing playbook.
