# make-your-own

## Objectives

You are free to choose the project idea you want to work on, based on your interests or passions (social media, blog, e-commerce, productivity tool, game, etc.).

However, your project must cover a maximum of professional skills required for the Application Designer and Developer role (Concepteur Développeur d'Applications - RNCP 37873).

You will need to demonstrate your ability to:

- Organize, model, and develop an application
- Design functional and technical models of your application (Merise and UML)
- Secure the application and its environment
- Work with project management methodologies (Kanban, backlog, task board)
- Prepare the deployment and documentation of the application

---

## Instructions

Your project must include:

- A **detailed analysis of needs** (functional and technical specifications)
- A complete **design phase**:
  - Functional models using **Merise method** (conceptual, logical, physical models)
  - **UML diagrams** (class diagram, sequence diagram at minimum)
  - **Mock-ups of user interfaces** (zoning, wireframes, design mockups)
- An **application architecture** organized in at least 3 layers:
  - **Presentation Layer** (frontend)
  - **Business Layer** (application logic)
  - **Data Access Layer** (connection to databases)
- The use of a **relational database** (SQL) and optionally a **NoSQL** database
- A **secured user authentication system** (registration/login with password hashing)
- Development of **responsive and accessible user interfaces**
- Development of **backend components** to handle business logic
- A **CI/CD pipeline** to automate tests and deployment
- **Version control** with a Git repository (with clear and meaningful commits)
- **Test plans**: unit tests or test scenarios (manual or automated)
- **Deployment preparation**:
  - Documentation for installation and configuration
  - Deployment files (Dockerfile, scripts, environment variables, hosting solution)
- **Project management**:
  - A simple backlog or task board
  - Versioning and organization of progress (Kanban or equivalent)

---

## Mandatory features

Your application must provide at least:

- **User account management** (signup, login, logout)
- **CRUD operations** on one or more main resources (e.g., posts, products, bookings)
- **Data persistence** with at least one SQL database
- **Protection against common security issues** (input validation, password hashing, basic authorization)
- **Error handling** (user-friendly messages, logging)
- **Technical documentation** including:
  - Project description
  - Installation and configuration instructions
  - Deployment preparation
  - Technologies used
  - Potential future improvements

---

## Suggested tools

- **Frontend**: HTML/CSS/JS, React, Vue, Angular...
- **Backend**: Node.js, PHP, Python, Go...
- **Databases**: PostgreSQL, MySQL, MariaDB, MongoDB...
- **Project management**: Trello, GitHub Projects, Notion...
- **Versioning**: Git, GitHub, GitLab...
- **Deployment**: Docker, Heroku, Vercel, Render, Railway...

---

## Bonus

You can add optional features to demonstrate more skills:

- Real-time functionality (chat, notifications with WebSocket)
- External API integration (OAuth, payments, maps, etc.)
- Administration panel (user roles)
- Multi-language support
- CI/CD with advanced deployment scenarios

---

## Deliverables

- Git repository containing:
  - Full source code
  - Database scripts or migrations
  - Mock-ups and diagrams (in a `/docs/` folder or linked)
  - README.md with documentation
- (Optional) A deployed version accessible online

---

## Evaluation criteria

| Category                  | Details |
| -------------------------- | ------- |
| Needs analysis & design     | Clear specifications, Merise models, UML diagrams, mock-ups |
| Application architecture   | Coherent, layered structure |
| Development quality        | Secure, functional, clean code, versioned |
| Data management            | Consistent data models, SQL used, access layer implemented |
| Testing                    | Present and relevant test plans |
| CI/CD & deployment         | CI/CD pipeline present, deployment ready |
| Project management         | Clear task organization, visible progress |
| Documentation              | Complete, clear, and professional |
| Overall presentation       | UX quality, creativity, performance |
