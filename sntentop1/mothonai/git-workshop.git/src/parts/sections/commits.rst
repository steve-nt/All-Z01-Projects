**************
Definition (?)
**************

A *“commit”* represents your commitement! Your integrity! One commit
means that you are commiting to the work you have done, the code (or
other stuff) you wrote in the repository.

Essentially, you are saying:

    *"yes! I made these changes and I am standing by them!"*

But how you are saying this?

**************
How to commit?
**************

::

   git commit -m "Makes dat feature work"

If you don’t provide the ``-m`` flag, your editor will open, presenting
you with the diff of your changes against the latest commit.
