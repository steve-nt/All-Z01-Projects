This would significantly simplify the simulation. Thus, the key is to find as many node-disjoint shortest paths as possible, assign ants to them, and then simulate their movement along these paths, one step per turn.

In this case, the total time would be (max number of ants per path -1) + path length.

Thus, the steps are:

1. Parse and validate input.

2. Check if a path exists from start to end. If not, error.

3. Find the shortest path length L.

4. Construct the subgraph of all nodes and edges that are part of any shortest path.

5. Find the maximum number of node-disjoint paths in this subgraph using flow algorithms.

6. Assign ants to these paths as evenly as possible.

7. Simulate the movement of ants along these paths, with each ant moving one step per turn. Since the paths are node-disjoint, there are no room or tunnel conflicts.

This would ensure the quickest movement.

Now, breaking down into implementation steps for Go:

2. **Building the Graph:**

- Represent rooms as nodes.

- Represent links as edges in an adjacency list.

3. **Shortest Path Check:**

- Perform BFS from start to end to find shortest path length L. If no path, error.

4. **Construct Subgraph of Shortest Paths:**

- For each node, compute distance from start (d_start) and distance to end (d_end).

- A node is in the subgraph if d_start + d_end == L.

- An edge a-b is in the subgraph if d_start[a] + 1 + d_end[b] == L or d_start[b] +1 + d_end[a] == L.

- This gives all nodes and edges that are part of any shortest path.

5. **Find Node-Disjoint Paths:**

- Use the flow network approach with split nodes to model node capacities.

- Create a new graph where each node (except start and end) is split into in and out nodes, connected by an edge of capacity 1.

- For edges in the subgraph, add directed edges with capacity 1.

- Compute maximum flow from start to end in this new graph. The max flow value is the number of node-disjoint paths.

6. **Extract Paths from Flow:**

- Decompose the flow into individual paths.

- Ensure each path has length L.

7. **Distribute Ants Among Paths:**

- Calculate how many ants to assign to each path to minimize the maximum (k_i -1 + L), where k_i is the number of ants in path i.

- This is done by dividing the ants as equally as possible among the paths.

8. **Simulate Movement:**

- For each ant, track its current position and path.

- For each turn, move each ant along its path if possible.

- Record the moves and print them in the required format.
