Fast-Forwarding
Fast-forwarding happens when the branch being merged into (e.g., main) has not diverged from the branch 
being merged (e.g., feature). In this case, Git simply moves the HEAD of the branch forward to the tip 
of the branch being merged, without creating a new merge commit.
Advantages:
Keeps history linear and clean.
Simple to execute if no divergent commits exist.
Fast-Forward Is Not Possible:
If the branches have diverged (i.e., both have unique commits since their common ancestor), 
Git cannot fast-forward, and a regular merge or rebase is required.

Merging
Merging combines the changes from one branch into another. It creates a merge commit to 
record the integration of changes, preserving the history of both branches.
Advantages:
Preserves a complete history of both branches.
Useful for maintaining branch histories in team projects.
Disadvantages:
History can become cluttered with merge commits, especially in long-lived branches with frequent merges.

Rebasing
Rebasing moves the commits of a branch to a new base (commit) to create a linear history. 
This involves replaying the commits from the branch being rebased onto the new base commit.
Advantages:
Linear and clean history.
Easier to follow when reviewing changes.
Disadvantages:
Alters commit history, which can cause issues when working with shared branches.
Should not be used on public or shared branches to avoid disrupting collaborators.

When to Use Which?
Fast-Forward: Use when no other commits exist on the target branch after branching off.
Merge: Use to preserve the history of both branches, especially in collaborative projects.
Rebase: Use to clean up commit history before merging, especially for feature branches or private branches.

changes and commit them, and you can discard any commits you make in this
state without impacting any branches by switching back to a branch.

If you want to create a new branch to retain commits you create, you may
do so (now or later) by using -c with the switch command. Example:

Or undo this operation with:

  git switch -
