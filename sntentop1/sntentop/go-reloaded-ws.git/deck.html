The Problem

Here's what your program sees, and what it must produce.

Input (sample.txt)

harold wilson (cap, 2) : ' I am a optimist ,but a optimist who carries a raincoat . '

↓

Output (result.txt)

Harold Wilson: 'I am an optimist, but an optimist who carries a raincoat.'

Audit case #4.

Let's Walk Through It

What happened between input and output?

1 (cap, 2) capitalised the 2 previous words: harold wilson → Harold Wilson
2 Punctuation : moved next to previous word: Wilson : → Wilson:
3 Quotes ' ... ' snapped to their content: ' I am ... raincoat . ' → 'I am ... raincoat.'
4 a became an before vowel: a optimist → an optimist
5 Commas and periods moved to the right place: ,but → , but    raincoat . → raincoat.

Five different functions, all in one sentence. Order matters.

All The Rules

7 transformation functions. All defined in the subject. All testable.

Numbers

(hex) (bin)
convert previous word to decimal

Casing

(up) (low) (cap)
all support , N to apply to N previous words (e.g. (up, 5))

Punctuation

. , ! ? : ;
stick to previous word, space after

Punct. Groups

... !?
stay grouped, no spaces inside

Quotes

' ... '
pairs wrap content, no inner spaces

Articles

a → an
before vowel or h

The tricky part: functions interact. We need a strategy.

Approach A: Pipeline

Process the text in stages. Each stage does one job.

Read
→
Tokenize
→
Markers
→
Punctuation
→
Quotes
→
a/an
→
Write

Pros

  • Test each function in isolation: (hex) doesn't touch punctuation
  • Print text between stages to see exactly where output went wrong
  • Easy to split work between teammates

Cons

  • Stage order is a real decision: (cap) before or after punctuation?
  • Stages need to agree on what a "token" looks like

Approach B: Finite State Machine

Scan left to right. Track where you are. Decide as you go.

Normal
Text
↔
Inside
Marker
↔
Inside
Quotes
↔
Punct.
Group

Pros

  • One pass, no intermediate data structures
  • State tells you exactly what context you're in: is this ' a quote opener or closer?

Cons

  • You handle (up), then discover (up, 2) needs a new transition — rewrite time
  • Wrong output? Good luck finding which state caused it

Which One?

Both can pass the project. The question is which one you can debug.

Pipeline

Same idea as chaining commands in your terminal.
The pattern you'll see most in your career.

FSM

How parsers, protocols, and embedded systems work.
The pattern when every byte matters.

Pick the one you can explain and test.
